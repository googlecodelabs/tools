En este proyecto vamos a poner en práctica el conjunto de comandos básicos de git que se han aprendido hasta el momento. Estaremos utilizando Github como la plataforma donde nuestro repositorio de trabajo podrá ser accedido.

Antes de comenzar

  • Asegúrese de tener una cuenta en Github y tener la sesión iniciada.
  • Asegurese de tener Git instalado.
  • Puede tener instalado algún editor de archivos. Una opción recomendada es usar Visual Studio Code para la edición de los archivos.

Primeros pasos

  • Ir a https://github.com/dalthviz/git_web_practice
  • Dar clic sobre el botón de
  • Ahora debería tener dentro de su cuenta un nuevo repositorio con el sitio web estático base con el nombre git_web_practice

Para el proyecto contemos con una Github App como herramienta para realizar algunas acciones sobre el repositorio del proyecto. Teniendo esto en cuenta, necesitamos que se le de acceso a está aplicación al repositorio previamente creado (git_web_practice)

Dando permisos a la Github App

  • Instalar la Github App disponible en https://github.com/apps/example-actions-bot
  • Clic
  • Seleccione su cuenta (donde quedó creado el repositorio git_web_practice)

  • Seleccione y seleccione el repostiorio git_web_practice. Clic en

Dado que ahora ya se configuro todo podemos empezar a trabajar. En este caso deberá clonar el repositorio creado. Para esto necesita abrir una terminal/consola con capacidad de ejecutar comando git. Para verificar si es posible puede probar con el comando git -v (debería con esto obtener la versión de git instalada).

Cloning...

  • En su repositorio git_web_practice de clic en
  • Copie la URL dada. Puede usar el botón (este botón copiara en su portapapeles la URL que se necesita).
  • En su terminal/consola escriba git clone <URL del repositorio>. Como ejemplo, el comando deberpia ser algo similar a git clone https://github.com/dalthviz/git_web_practice.git (si su usario en github fuera dalthviz).
  • Cambie el directorio activo (cd)a la carpeta raíz del repositorio. Ejecutando cd git_web_practice debería bastar.

Ahora que tenemos una copia local del repositorio podemos hacer los cambios necesario para corregir los errores. En este caso el repositorio contiene una página web estática que puede revisar abriendo los archivos en su navegador. La idea general es arreglar este sitio según los arreglos listados a continuación. Por cada arreglo especificado es necesario hacer un commit. Además, luego de cada commit, debe subir sus cambios locales mediante un push al repositorio remoto.
